# 上市公司研究 AI 工作流

一套面向上市公司公开资料的结构化研究工作流，适用于 Codex 等能够读取本地资料、处理表格和执行审计的 AI 编程代理。

当前版本：**V5.9**（2026-06-14）

## 它解决什么问题

- 递归盘点年报、季报、业绩稿、电话会、IR 材料等来源，减少资料已存在但未进入数据表的静默遗漏。
- 建立逐文件状态、文件指纹和反向覆盖审计，使资料库、原始数据长表与展示表可以对账。
- 分别验收电话会 prepared remarks 与 Q&A，识别目录页、登录页、短页壳等不完整文件。
- 构造预期披露期间全集，主动发现尚未采集的季度或年度资料。
- 生成精简、可追溯的 Excel 数据底稿，并保留本地文件与公网来源双通道链接。
- 跟踪管理层明确提出的目标和指引，对已经到期的目标进行证据化验证。

## 下载

打开 GitHub 仓库后，点击 **Code → Download ZIP** 即可下载全部文件。GitHub 的 Releases 页面发布版本后，也可以直接下载对应版本的压缩包。

## 安装为 Codex Skill

1. 下载或克隆本仓库。
2. 将整个目录复制到 Codex skills 目录，例如：

```text
%USERPROFILE%\.codex\skills\company-research
```

3. 确认该目录下直接存在 `SKILL.md`。
4. 重新打开 Codex 会话，在研究上市公司时要求使用 `company-research` skill。

任务清单是主指令的配套执行规范。需要完整复核流程时，请同时向 AI 提供 `研究工作流任务清单V5.9.md`。

## 使用边界

- 本工作流用于资料整理、数据抽取和证据审计，不构成投资建议。
- 自动抽取结果仍需人工复核，尤其是期间归属、单位换算、非 GAAP 口径和管理层指引。
- 本仓库不包含任何公司资料库、付费电话会文本、财务数据文件或第三方受版权保护的原始材料。
- 使用者应自行确认其数据来源、下载方式和再分发行为符合适用的条款与法律。

## 已知限制

V5.9 已显著加强资料覆盖闭环，但仍建议后续继续完善：

- 将预期期间审计严格扩展为“期间 × 必需来源类型”。
- 统一不完整文件、部分解析等状态定义。
- 为所有来源类型设置最低抽取覆盖要求，而不只关注电话会。
- 提供可直接运行的确定性审计脚本和明确的密度阈值配置。
