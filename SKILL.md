# Company Research Skill V5.9

默认输出语言为中文。Ticker、官方文件名、来源标题、英文指标名、原文摘录保留原文。

V5.9 是“资料零遗漏(含未采集缺口) + 可复跑覆盖审计 + 可迁移双通道链接 + 精简展示数据底稿 + CEO目标兑现追踪版”。目标不是生成庞大的资料库，也不是整理 CEO 言论，而是稳定产出一份干净、可读、可追溯、可核验的公司数据工作簿；资料库中的每个候选文件都必须有处理结果，每个来源链接都必须能在交付环境中定位或明确标注失效原因。

## 1. V5.9 核心变化

1. 仍然取消宽泛 CEO 言论功能：
   - 不生成 CEO 言论时间线。
   - 不生成 CEO Word。
   - 不生成 CEO 言论明细 sheet。
   - 不按主题整理管理层观点。
2. 新增窄口径 `CEO目标兑现追踪` sheet：
   - 只收录 CEO/管理层明确提出的目标、指引、预期、承诺。
   - 已到期目标必须连接 `最终展示数据`、`原始数据长表`、年报/20-F、季报/6-K、后续电话会等来源验证。
   - 未到期目标/指引/预期单独列为观察清单，由用户自行跟踪，不做兑现判断。
   - 禁止把普通 CEO 观点、战略叙事、无验证口径的表述塞入该 sheet。
3. 电话会仍然必须读取：
   - 取消 CEO 言论产品，不等于跳过电话会。
   - 电话会 prepared remarks 和 Q&A 中披露的一切经营业务数字，都必须进入结构化流程。
4. Excel 默认保留 4 个 sheet：
   - `使用说明`
   - `最终展示数据`
   - `原始数据长表`
   - `CEO目标兑现追踪`
5. 年度数据和季度数据不横向混接：
   - 年度分块使用 `FY2019`、`FY2020` 等列。
   - 季度分块使用 `2019Q1`、`2019Q2` 等列。
   - 年度和季度因来源、口径、披露节奏不同，必须在同一 sheet 中分块展示。
6. 业务及经营相关数据统一称为“业务数据”：
   - 使用 `年度业务数据`、`季度业务数据`。
   - 不再混用“运营数据”“业务数据”“Operating metrics”等多个中文口径。
7. 新增逐文件资料覆盖闭环：
   - 抽取前递归盘点资料库，生成确定性的文件清单和文件指纹。
   - 每个候选文件必须落入 `已抽取`、`已读取无可用数据`、`重复文件`、`无法解析`、`期间无法识别`、`非研究材料` 之一，并填写原因。
   - 可解析资料存在但在长表中零记录、且没有合规原因时，禁止交付。
8. 电话会按 prepared remarks 与 Q&A 双区验收：
   - 两个区块分别记录是否存在、是否完整、是否已抽取。
   - 只有目录、参会人、导航、登录提示或短页壳的文件不得当作完整 transcript。
9. 新增期间、来源和引文一致性校验：
   - 文件名期间、文档标题/日期、长表期间必须一致。
   - 数字记录的原文依据必须包含或明确支持该数字与指标。
   - 禁止把上一季度文件误挂到下一季度数据，除非该行明确是跨期比较并写明口径。
10. 链接改为可迁移双通道：
   - 本地链接使用 `资料库根目录参数 + 资料库相对路径` 动态生成。
   - 公网链接优先使用季度/文件级永久链接，作为本地链接的备用通道。
   - 禁止写入 staging、cache、temp、Downloads 或其他临时目录的绝对路径。
11. 新增预期披露期间全集对账（关闭“本就未采集”的缺口）：
   - 从公司上市/首个可得财报期推导到最新已报告期，按财年与季度构造预期期间全集。
   - 预期全集必须与资料库文件清单、原始长表三方对账，每个期间标记 `资料齐备`、`资料库缺口(未采集)`、`已确认无此期间`。
   - `资料库缺口(未采集)` 必须触发官方 IR/SEC/权威 transcript 补采；补采失败须显式登记 `外部不可得`，不得静默通过。
   - `季度缺口扫描表` 由可选产物提升为强制交付物。
12. 新增抽取密度下限与跨季合理性：
   - 在相对阈值之外增加绝对下限：解析成功且含 prepared remarks 与 Q&A 的完整 transcript，业务/财务记录数或覆盖密度低于绝对下限时必须复核或说明。
   - 覆盖摘要记录每份来源的覆盖密度，用于发现“全季均匀偏薄”，相对阈值无法识别的情形。
13. 覆盖审计必须可复跑并设交付门禁：
   - 逐文件覆盖审计、电话会双区审计、预期期间全集对账、链接可达性审计由确定性脚本产出，附脚本与输出日志，重复运行结果一致。
   - 任一阻断项未消解、自检不通过，禁止交付。

## 2. AI 与用户边界

AI 负责：

- 扫描本地目录和用户指定 Drive 目录。
- 为每个候选文件建立文件指纹、相对路径、期间、解析状态和抽取状态。
- 下载或补齐权威来源。
- 读取年报、季报、业绩稿、电话会 transcript、Q&A、IR 材料。
- 抽取财务数据、固定年度数据、人效数据、股本数据、所有业务数据。
- 建立 source_id、原始长表和展示表回链。
- 计算年度同比增长率、年度业务绝对增减、季度业务环比净增。
- 建立 CEO/管理层明确目标的提出证据、验证证据和兑现状态。
- 标注未披露、不可比、口径变化和待核实项。
- 执行逐文件反向覆盖审计、电话会双区审计和链接可达性审计。

AI 不做：

- 不替用户形成投资判断。
- 不宣称公司有无壁垒。
- 不整理 CEO 言论时间线、逻辑线或观点摘录，除非用户单独要求。
- 不对未到期目标、定性预期做“改善、承压、待观察”等主观判断。
- 不把战略叙事、管理层观点、没有数字的表述塞进数据表。

## 4. 默认工作簿结构

V5.9 工作簿只保留：

### 4.1 `使用说明`

必须包含：

- 公司、ticker、数据范围。
- 资料覆盖范围：年报、季报/6-K、业绩稿、电话会 transcript。
- 来源优先级。
- 单位规则。
- 年度/季度分块规则。
- 主表筛选规则。
- `资料库根目录`：可编辑的绝对路径参数，并定义为 Excel 命名区域 `资料库根目录`；根目录迁移后只需更新这一处。
- 资料覆盖摘要：候选文件数、已解析数、已抽取数、零抽取文件数、无法解析数、重复文件数、缺失期间。
- 电话会覆盖摘要：各期间 prepared remarks 与 Q&A 的存在、完整和抽取状态。
- 链接验证摘要：展示表回链、本地文件链接、公网链接的已验证数、失败数和失败原因。

### 4.2 `最终展示数据`

唯一主展示页。必须纵向分块，不得把年度列和季度列混成一张横向大表。

固定分块顺序：

1. 年度通用财务数据
2. 年度人效数据
3. 年度股本与资本结构数据
4. 年度业务数据
5. 季度财务数据
6. 季度业务数据

如某公司没有某分块数据，可保留分块标题并写 `未披露`，或在 `使用说明` 中说明不展示该分块。

### 4.3 `原始数据长表`

必须包含：

| 字段 | 要求 |
|---|---|
| source_id | 稳定唯一 ID；由 ticker、期间、来源文件指纹、区块、指标等稳定字段生成，不得只依赖易变化的行号 |
| file_id | 文件级稳定 ID；优先由规范化相对路径 + 文件大小 + SHA-256 生成 |
| 来源类型 | 年报、季报、业绩稿、电话会 prepared remarks、电话会 Q&A、IR deck 等 |
| 期间 | FY2025、2025Q4 等 |
| 期间类型 | 年度 / 季度 |
| 数据归类 | 财务数据 / 人效数据 / 股本与资本结构数据 / 业务数据 |
| 指标名称 | 中文 + 英文 |
| 原始指标 | 原文披露字段或原句中的指标名 |
| 标准化数值 | 进入展示表的标准化数值 |
| 单位 | 统一后的单位 |
| 原始数值 | 原文数值 |
| 原始单位 | 原文单位 |
| 来源文件 | 规范文件名或网页标题，不得写临时目录绝对路径 |
| 资料库相对路径 | 相对于 `资料库根目录` 的稳定路径，例如 `TIGR/04_Earnings_Calls/2025_q4.html` |
| 本地文件链接 | 由根目录参数与相对路径动态生成的可点击链接 |
| 来源URL | 季度/文件级公网永久链接；通用 transcript 首页不得作为唯一来源链接 |
| 来源链接状态 | `双通道可用`、`仅本地可用`、`仅公网可用`、`均不可用` |
| 文件指纹 | 文件大小和 SHA-256，支持文件移动后的匹配与去重 |
| 位置 | 页码、表格、段落、speaker、Q&A 编号等 |
| 口径/原文依据 | 简短说明 |
| 是否进入展示表 | 是 / 否 |
| 未进入原因 | 低层附注、不可归类、重复来源、非业务数字等 |

`本地文件链接` 不得保存旧机器或临时抓取目录的硬编码路径。推荐公式逻辑：

```text
=HYPERLINK("file:///" & SUBSTITUTE(资料库根目录 & "\" & 资料库相对路径, "\", "/"), "打开本地文件")
```

如果工作簿移动或资料库根目录变化，只更新 `使用说明` 中的根目录参数，不重写全部长表链接。

### 4.4 `CEO目标兑现追踪`

该 sheet 只用于追踪 CEO/管理层明确目标、指引、预期和承诺，不得变成 CEO 摘录表。

必须分为两区：

#### A. 已到期 / 可验证 CEO目标兑现追踪

字段必须包含：

| 字段 | 要求 |
|---|---|
| 目标ID | 稳定唯一 ID，例如 `T-2024-PC` |
| 提出期间 | 例如 `2024Q2`、`FY2024` |
| 提出者 | CEO、CFO、管理层；无法确认时写 `管理层` |
| 目标/指引原文摘要 | 简洁摘录目标内容，不粘贴长篇原文 |
| 指标标准名 | 例如净新增付费客户、净新增入金账户、收入指引 |
| 验证口径 | 例如 `FY2024 - FY2023`、`2025Q4期末值` |
| 目标期限 | 例如 `FY2024`、`2026及以后` |
| 目标值 | 数值目标；定性目标写明性质 |
| 实际值 | 到期后的实际披露值或按主表可验证数据计算值 |
| 超额/缺口 | `实际值 - 目标值`，不适用时写 `N/A` |
| 完成率 | `实际值 / 目标值`，不适用时写 `N/A` |
| 客观结论 | 只允许：超额兑现、已兑现、未兑现、部分兑现、缺少可验证披露、不可量化目标 |
| 验证来源 | 必须指向 `最终展示数据`、`原始数据长表` 或后续公开材料 |
| 提出时原文证据 | 简短原文 |
| 提出来源URL | 来源链接 |

已到期目标验证不得只查 CEO sheet。必须优先尝试连接：
1. `最终展示数据` 中已标准化的年度/季度业务数据。
2. `原始数据长表` 中同指标、同期间或后续期间的记录。
3. 年报 / 20-F / 10-K。
4. 季报 / 6-K / 10-Q / EX-99。
5. 后续业绩稿和电话会。

#### B. 未到期 CEO指引 / 预期观察清单

字段必须包含：

| 字段 | 要求 |
|---|---|
| 观察ID | 稳定唯一 ID，例如 `P-2026-FA` |
| 提出期间 | 指引/预期提出时间 |
| 提出者 | CEO、CFO、管理层 |
| 未到期指引/预期 | 简洁摘录 |
| 观察指标 | 未来应观察的指标 |
| 目标期间 | 未来验证期间 |
| 目标值/性质 | 数值或定性性质 |
| 状态 | 固定写 `未到期` |
| 未来验证方法 | 未来应如何用主表或原始数据验证 |
| 原文证据 | 简短原文 |
| 来源URL | 来源链接 |

未到期区禁止计算完成率，禁止写“改善、承压、待观察、趋势较好”等判断性词语。只列事实和未来验证方法。

## 5. 来源优先级

同一指标、同一期间存在多个来源时，按以下优先级选入展示表：

1. 官方年报 / 20-F / 10-K
2. 官方季报 / 6-K / 10-Q / EX-99
3. 官方业绩新闻稿
4. 电话会 prepared remarks 中管理层披露的数据
5. 电话会 Q&A 中管理层披露的数据
6. IR deck / Investor Day
7. 监管或交易所公告
8. 权威新闻稿分发源
9. 第三方 transcript

电话会中的业务数字不能因来源优先级较低而被忽略。来源优先级只用于解决同一指标同一期间的冲突，不用于跳过电话会业务数据抽取。

### 5.1 抽取前逐文件盘点

开始抽取前，必须递归扫描公司资料库及用户指定的云端目录。至少覆盖年报、季报/6-K/10-Q、业绩稿、电话会、IR 材料、监管公告和用户补充材料所在目录。

`全部资料清单.xlsx` 必须为每个候选文件记录：

| 字段 | 要求 |
|---|---|
| file_id | 稳定文件 ID |
| 文件名 | 原始文件名 |
| 资料库相对路径 | 相对于公司资料库根目录 |
| 文件类型 | PDF、HTML、TXT、DOCX、XLSX 等 |
| 资料类别 | 年报、季报、业绩稿、电话会、IR deck 等 |
| 识别期间 | FY2025、2025Q4 等 |
| 文件大小 | 字节数 |
| SHA-256 | 内容指纹，用于去重和移动后识别 |
| 解析状态 | 成功、部分成功、失败 |
| 抽取状态 | 已抽取、已读取无可用数据、重复文件、无法解析、期间无法识别、非研究材料 |
| 长表记录数 | 该文件对应的原始长表记录数 |
| 未抽取原因 | 状态不是 `已抽取` 时必填 |
| 链接状态 | 本地、公网和双通道状态 |

禁止只记录“搜到多少个文件”。清单文件数必须能与资料库实际候选文件数逐一核对，且同一文件移动目录后能通过指纹识别。

### 5.2 零静默遗漏验收

抽取完成后必须执行 `资料库文件 -> 文件清单 -> 原始数据长表 -> 最终展示数据` 的反向审计：

1. 每个候选文件都必须出现在文件清单。
2. 每个可解析且含研究数据的文件必须至少对应一条长表记录。
3. 零长表记录文件必须有明确、可复核的状态和原因。
4. 同一期间存在多个来源时，可因重复来源不进入长表，但必须在文件清单中指向保留的 `file_id`。
5. 文件清单中识别出的期间集合必须与长表按来源类型的期间集合对账。
6. 任一可解析文件静默缺失、任一期间无解释缺口，均为交付阻断项。

`资料覆盖摘要.txt` 至少写明：候选文件总数、成功解析数、部分解析数、解析失败数、已抽取文件数、零抽取文件数、重复文件数、缺失期间，以及每个异常文件的相对路径和原因。

### 5.3 电话会文件完整性与双区覆盖

每份电话会文件必须分别审计 prepared remarks 和 Q&A：

| 检查项 | 合格要求 |
|---|---|
| 文本完整性 | 不是只有标题、目录、参会人、导航、登录提示、页脚或短页壳 |
| prepared remarks | 标记 `已抽取`、`原文无该区块` 或 `文件不完整` |
| Q&A | 标记 `已抽取`、`原文无Q&A` 或 `文件不完整` |
| speaker/位置 | 长表记录能够定位 speaker、段落或问答编号 |
| 期间 | 文件名、标题/日期与长表期间一致 |

短文本阈值只能作为预警，不得作为唯一判断。若文件明显不完整，必须尝试官方 webcast/transcript、SEC 附件、权威 transcript 或其他完整版本；仍无法补齐时，在覆盖摘要中列为 `文件不完整`，不得假装该季度已完整覆盖。

### 5.4 期间、指标与引文一致性

- 从文件名、文档标题、发布日期和正文财报期间独立识别期间；发生冲突时先人工/语义复核，不得直接写入长表。
- 长表 `期间` 必须与来源期间一致。跨期比较行必须在口径中明确写出比较对象。
- `口径/原文依据` 必须包含或明确支持该指标与数值，不能用无关段落充当证据。
- 每条数字记录必须检查原始数值、单位、符号和标准化结果。
- 同一季度行数显著低于相邻季度或历史中位数的 30% 时触发人工复核；它是异常预警，不是自动删行规则。

### 5.5 预期披露期间全集对账

逐文件盘点只能保证“库里有的不丢”，不能保证“本该有的都在”。抽取前必须独立构造预期披露期间全集：

- 从公司上市日期或首个可得财报期，推导到最新已报告期，按财年（FY）与季度（Q）列出应存在的报告期序列。
- 预期全集必须同时与 `全部资料清单.xlsx` 期间集合、原始长表期间集合三方对账。
- 每个预期期间只能落入 `资料齐备`、`资料库缺口(未采集)`、`已确认无此期间(如上市前)` 之一。
- `资料库缺口(未采集)` 必须触发补采：依次尝试官方 IR、SEC 文件、官方 webcast/transcript、权威 transcript；成功后回到逐文件流程，确实无法获取时在覆盖摘要显式登记 `外部不可得` 及尝试记录。
- `季度缺口扫描表` 为强制产物，随交付物一并生成，记录每个预期期间的三方状态与原因。
- 任一预期期间为 `资料库缺口(未采集)` 且未登记补采结论，为交付阻断项。

### 5.6 抽取密度下限与跨季合理性

`5.4` 的 30% 相对阈值无法发现“所有季度都均匀偏薄”，因此追加绝对密度校验：

- 对解析成功、且含 prepared remarks 与 Q&A 的完整 transcript，定义覆盖密度指标（每千字记录数，或每发言段/每问答轮记录数）。
- 设定绝对下限；任一完整 transcript 的业务/财务记录数或覆盖密度低于绝对下限时，必须人工复核或在覆盖摘要写明合规原因（如该季确实披露稀少）。
- 同一公司各季覆盖密度的离散度，以及与同类来源、同行可比工作簿的密度差异，均作为合理性预警。
- 密度校验是复核触发器，不是自动补全或自动删行规则；但“整体偏薄且无说明”视为交付阻断项。
- `资料覆盖摘要.txt` 必须记录每份来源的覆盖密度与触发的预警。

### 5.7 覆盖审计可复跑与交付门禁

覆盖闭环不能只是人工声明，必须可复跑、可复核：

- 逐文件反向覆盖审计、电话会 prepared remarks/Q&A 双区审计、预期期间全集对账、链接可达性审计，均由确定性脚本产出，附脚本与输出日志。
- 同一资料库与工作簿重复运行审计，结果必须一致；文件移动后通过 SHA-256 与文件大小重新定位仍能复现。
- 交付前必须跑通自检门禁：任一阻断项（静默缺失文件、零记录无原因、预期期间未采集未登记、完整 transcript 双区缺读、错季挂链、不可达链接标为可用）未消解，自检不通过，禁止交付。

## 7. 业务数据抽取硬规则

业务数据不预设固定指标。业务数据必须依据资料逐项抽取。

业务数据包括但不限于：

- 用户、客户、账户、入金客户、付费客户、活跃用户。
- 新增客户、新增付费用户、新增入金客户。
- 客户资产、客户净流入、AUM、资产余额。
- 交易额、交易量、订单数、合约数、换手、交易频次。
- 地区业务数据，例如美国、香港、新加坡、马来西亚、日本等地区的客户、资产、交易、收入。
- 产品业务数据，例如期权、期货、财富管理、融资融券、基金、加密、企业服务等。
- 获客、留存、转化、活跃、ARPU、CAC、payback。
- 管理层披露的目标、指引、完成度。
- 市场份额、排名、牌照覆盖、产品覆盖等可量化事实。

### 7.1 电话会业务数据不可遗漏

电话会 prepared remarks 和 Q&A 中，只要出现经营业务方面的数据，都必须抽取到 `原始数据长表`。

例如：

- 本季度美国新增多少付费用户。
- 本季度马来西亚客户资产增加多少。
- 某地区客户净流入、客户资产、交易额、开户数。
- 某产品新增用户、交易额、AUM、余额。
- 管理层披露某项业务达到多少规模、环比/同比增长多少。

这些数据即使没有出现在年报或季报表格中，也必须进入 `季度业务数据` 分块；如果只能归属年度，则进入 `年度业务数据` 分块。

遗漏电话会业务数据，视为 V5.9 交付不合格。prepared remarks 或 Q&A 任一区块未读取且没有 `原文无该区块` 或 `文件不完整` 说明，同样视为不合格。

## 8. 展示表行规则

### 8.1 财务、人效、股本数据

实际数值类指标展示两行：

1. 原始值 / Reported value
2. 同比增长率 / YoY growth

本身就是百分比、占比、率、margin、ratio、as % 的指标只展示原始值，不再机械增加同比增长率行，也不再增加“百分比的增长百分比”。

百分比类指标必须按百分比格式展示，例如 `45.1%`，不得显示为 `45.1` 或 `0.451` 这类不清楚口径的数值。

### 8.2 年度业务数据

每个业务指标展示三行：

1. 原始值 / Reported value
2. 同比增长率 / YoY growth：`本年值 / 上年值 - 1`
3. 绝对值增减 / Absolute change：`本年值 - 上年值`

如果年度业务指标本身就是百分比、占比、率、margin、ratio、as %，不生成同比增长率行；是否生成绝对值增减行应以 `百分点 / ppt` 口径表达，且必须能解释研究意义。

### 8.3 季度业务数据

每个业务指标展示两行：

1. 原始值 / Reported value
2. 环比净增 / QoQ net add：`本季度值 - 上季度值`

季度业务数据不默认生成 `同季同比 / YoY growth` 行。只有用户单独要求季度业务同比分析时，才另行增加。

如果公司直接披露“本季度净新增 X”，优先保留公司披露值；如果同时能由期末值推算，应在口径中注明用于核对。

### 8.4 缺失与不可比

- 缺少基期时，年度增长率、绝对值增减、环比净增写 `N/A`。
- 口径变化时写 `不可比`，并在口径说明中写明原因。
- 未披露时写 `未披露`。
- 不允许空白单元格无说明。

## 9. 主表筛选硬规则

`原始数据长表` 可以全量收纳，但 `最终展示数据` 必须是精选展示层。

允许进入主表：

- 固定年度通用财务数据。
- 固定年度人效数据。
- 固定年度股本与资本结构数据。
- 材料中披露的业务数据。
- 对投资分析有明确用途的公司特有指标。

禁止进入主表：

- 20-F 附注里的低层应收、应付、税项、员工往来、VIE 明细。
- 只出现一次、无法形成序列、且无明确研究价值的细碎科目。
- 机器关键词扫到但无法通过语义归类的数据。
- 没有数字的管理层观点或战略叙事。
- `待人工定义`、`待核实`、`待核对`、`无法核实`、`待确认` 等占位值。
- 内部链接文本、跳转地址或公式引用文本，例如 `#'原始数据长表'!A123`。

如果某个指标只有占位值、未披露值或无法核实值，整行不得进入 `最终展示数据`。如果某个期间无法核实，该期间不得填入占位文本。

主表规模要求：

- 固定财务、人效、股本分块应尽量控制在 40-80 个原始指标。
- 业务数据分块不设固定行数，但必须归类清楚。
- 不设置固定行数硬性红线。`最终展示数据` 可以根据材料披露情况扩展，但必须保持分组清晰、顺序稳定、排版可读。
- 如果展示表明显过长，应在 `使用说明` 中说明原因，并用分块、冻结窗格、筛选和合理列宽保证可读性。

## 10. 单位规则

- 金额统一使用“亿”口径，例如亿美元、亿港元、亿人民币。
- 费用、成本、支出类金额在 `最终展示数据` 中统一按负值展示，即使公司原表用正数披露，也要转为支出负值，并在口径说明中标注“费用/成本按支出负值展示”。
- 人数可使用人、万人、千人，但同一指标序列必须统一。
- 账户数、客户数、用户数同一指标序列必须统一。
- 百分比保留 `%`。
- 百分比识别必须依据明确口径：单位为 `%`，或指标含“占比/比例/利润率/毛利率/净利率/负债率”，或英文独立词 `margin`、`ratio`、`rate`。不得用无边界子串匹配把 `operations` 误判为 `ratio`。
- 百分点变化使用 `百分点` 或 `ppt`。
- 股数统一为亿股、百万股或 ADS，必须清楚标注。
- 对 ADR/ADS 公司，股本数据必须明确是普通股、ADS 还是 ADS 等效口径；如果公司披露普通股数且 1 ADS 对应多股普通股，展示表应同时给出普通股口径和 ADS 等效口径，并写明换算比例。
- 同一张原始表含混合单位时，行级语义优先于表级单位。

## 11. 链接验证硬规则

`最终展示数据` 中每个原始值数据格必须链接到 `原始数据长表` 对应 source_id 行。

### 11.1 工作簿内部回链

必须验证：

- 链接存在。
- 链接目标行存在。
- 展示表数值与原始长表标准化数值一致。
- 指标语义一致。
- 单位一致或有明确换算。
- 派生行不得继承原始数据超链接。

### 11.2 本地来源链接

- `使用说明` 设置一个可编辑的 `资料库根目录` 参数，并将该单元格定义为同名 Excel 命名区域。
- 长表只保存规范文件名和 `资料库相对路径`，本地可点击链接由根目录参数动态拼接。
- 交付前逐条检查目标文件是否存在，禁止把不存在的路径显示为可用链接。
- 相对路径必须规范化，禁止包含 `..` 跨出资料库根目录。
- 禁止出现 `_upload_staging`、`_from_drive`、`_cache`、`temp`、`tmp`、Downloads 或一次性下载目录路径。
- 文件发生移动时优先通过 SHA-256 和文件大小重新定位，并更新相对路径；不得保留失效旧路径蒙混过关。

### 11.3 公网来源链接

- 优先使用官方文件页、SEC 文件页、官方 IR 文件或季度/文件级 transcript 永久链接。
- 通用 transcript 首页、搜索结果页或需要再次手工搜索的页面不得作为唯一来源链接。
- 交付前验证 HTTP 成功响应、重定向后的最终地址和页面内容相关性。登录墙或权限限制必须标注，不能写成 `可用`。
- 公网链接失效但本地文件可用时标记 `仅本地可用`；本地缺失但公网可用时标记 `仅公网可用`；两者均可用时标记 `双通道可用`。

链接验证摘要写入 `使用说明`，至少包括：展示表内部回链、本地文件链接、公网链接、双通道链接、失效链接和失效原因。任何内部回链错误、任何被标记为可用但实际无法打开的链接、任何无解释的 `均不可用`，均不得交付。

配色必须克制、统一、清晰，不得五颜六色。

## 13. 完成前检查

交付前必须确认：

- 工作簿只有 `使用说明`、`最终展示数据`、`原始数据长表`、`CEO目标兑现追踪`，除非用户另有要求。
- 资料库候选文件数与 `全部资料清单.xlsx` 文件数已逐一对账。
- 每个候选文件都有抽取状态；不存在静默消失的文件。
- 所有可解析且含研究数据的文件均有长表记录；零抽取文件均有可复核原因。
- `资料覆盖摘要.txt` 已列明缺失期间、解析失败、部分解析、重复文件和零抽取文件。
- 没有 CEO 言论时间线、CEO Word、CEO 明细、电话会问答等宽泛 CEO/电话会摘录 sheet。
- 电话会 transcript 已被读取并抽取业务数据。
- 每份电话会的 prepared remarks 与 Q&A 已分别验收；短页壳没有被误判为完整 transcript。
- 电话会各季度覆盖已与资料库文件清单对账，不存在资料库有文件但长表无记录的季度。
- 电话会中的业务数字没有被当作“CEO 言论”而跳过。
- `CEO目标兑现追踪` 只包含明确目标、指引、预期或承诺，不包含普通管理层观点摘录。
- 已到期 CEO 目标已连接主表、原始长表或后续公开材料验证，不得写“无法验证”而未查主表。
- 未到期 CEO 指引/预期已单独列入观察清单，状态固定为 `未到期`，不计算完成率。
- `CEO目标兑现追踪` 中不得出现“改善、承压、待观察、趋势较好”等主观状态词。
- 年度和季度在展示表中分块展示。
- 业务数据统一称为业务数据。
- 人效数据进入年度主表。
- 年度业务指标有同比增长率和绝对值增减；季度业务指标有环比净增，不默认生成季度业务同比。
- 百分比、占比、率类指标不机械生成增长率行，且按百分比格式展示。
- 费用、成本、支出类金额已统一用负值展示，营业利润等金额类指标不得被误设为百分比单位或百分比格式。
- 无法核实或只是占位的数据没有进入最终展示表。
- 主表没有机械灌入 20-F 附注低层科目。
- 所有原始值数据格可回链到原始数据长表。
- 长表期间与文件名、标题/日期和正文期间一致，原文依据与指标数值一致。
- 本地链接使用根目录参数和相对路径，不含 staging/cache/temp 等临时绝对路径。
- 本地文件链接和公网链接已逐条验证，状态与实际一致。
- 预期披露期间全集已构造，并与文件清单、长表三方对账；`季度缺口扫描表` 已生成。
